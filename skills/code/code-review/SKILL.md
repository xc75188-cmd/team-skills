# Code Review Skill

## 用途

对代码变更（diff 或完整文件）进行结构化审查，从正确性、安全性、可维护性、性能四个维度给出反馈。适用于 PR review 或自查。

## 使用方法

将代码 diff 或文件内容提供给 Agent，触发此 Skill 即可。

## 输入

- 代码 diff 或完整源文件
- （可选）项目上下文和编码规范

## 输出

- 按严重程度分级的问题清单
- 每个问题的具体改进建议
- 总体评价

## 指令

1. 通读所有代码变更，理解整体改动意图
2. 从以下四个维度逐一检查：
   - **正确性**：逻辑是否正确，边界条件是否处理，是否有 bug
   - **安全性**：是否有 SQL 注入、XSS、敏感信息泄露等风险
   - **可维护性**：命名是否清晰，结构是否合理，是否有重复代码
   - **性能**：是否有明显的性能问题（N+1 查询、不必要的循环等）
3. 将发现的问题按严重程度分为三级：
   - 🔴 必须修复：影响正确性或安全性的问题
   - 🟡 建议修改：影响可维护性或有潜在风险
   - 🟢 可选优化：代码风格、微小改进
4. 对每个问题给出具体的修改建议，而不仅仅是指出问题
5. 最后给出总体评价和是否建议合并的结论

## 注意事项

- 审查时应结合项目上下文，避免脱离实际场景的建议
- 不要对代码风格偏好做过于主观的评判
- 如果变更量很大，优先关注核心逻辑的变更
