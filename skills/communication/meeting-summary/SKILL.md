# Meeting Summary Skill

## 用途

将会议录音转录文本或原始笔记整理为结构化的会议纪要，提取关键决定、待办事项和后续跟进。

## 使用方法

将会议转录文本或笔记提供给 Agent，触发此 Skill 即可。

## 输入

- 会议转录文本（来自 Otter、飞书妙记、腾讯会议等）
- 或手动记录的会议笔记

## 输出

结构化的会议纪要，包含以下部分：

- 会议基本信息
- 议题与讨论要点
- 关键决定
- 待办事项（含负责人和截止日期）
- 后续跟进

## 指令

1. 通读全部会议内容，识别参与者和讨论的主要议题
2. 按以下格式输出会议纪要：

```
## 会议信息
- 日期：
- 参会人：
- 时长：

## 讨论要点
按议题分段总结，每个议题 2-3 句话概括核心观点和讨论。

## 关键决定
列出会议中明确做出的决定，每条用一句话描述。

## 待办事项
| 事项 | 负责人 | 截止日期 |
|------|--------|----------|
| ... | ... | ... |

## 后续跟进
下次会议时间、需要进一步讨论的议题等。
```

3. 如果转录文本中有不清楚的内容，标注 [待确认] 而不是猜测
4. 待办事项必须明确负责人，如果转录中未提及，标注 [待分配]

## 注意事项

- 优先保留决策和行动项，闲聊和重复讨论可以省略
- 保持客观中立，不要添加转录中没有的内容
- 如果会议很长（超过1小时），在开头加一个 3-5 句话的摘要
